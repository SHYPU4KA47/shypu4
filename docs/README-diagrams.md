# UML Диаграммы для отчёта

## Как использовать

### Вариант 1: Онлайн-конвертация
1. Откройте файлы `.puml` в любом текстовом редакторе
2. Скопируйте содержимое
3. Вставьте на один из сайтов:
   - http://www.plantuml.com/plantuml/uml/ (вставьте код и получите PNG)
   - https://plantuml-editor.kkeisuke.com/ (редактор с предпросмотром)

### Вариант 2: IntelliJ IDEA
1. Установите плагин "PlantUML integration"
2. Откройте файл `.puml`
3. Нажмите правой кнопкой → "Export Current Diagram" → выберите PNG/SVG

### Вариант 3: VS Code
1. Установите расширение "PlantUML"
2. Откройте файл `.puml`
3. Нажмите Alt+D для предпросмотра
4. Экспортируйте через меню

## Размещение в отчёте

### UML диаграмма классов
**Раздел: 1.3 Проектирование архитектуры приложения**
**Подраздел: 1.3.1 UML диаграмма классов**

Диаграмма показывает структуру классов приложения:
- `OpenLibraryClient` - основной класс с методами поиска
- `SearchResponse` и `Doc` - классы для хранения данных от API
- `BookSearchGUI` - графический интерфейс (если используется)
- `OpenLibraryClientTest` - класс для тестирования

### UML диаграмма прецедентов
**Раздел: 1.3 Проектирование архитектуры приложения**
**Подраздел: 1.3.2 UML диаграмма прецедентов**

Диаграмма показывает взаимодействие пользователя с системой:
- Ввод поискового запроса
- Выполнение поиска
- Просмотр результатов
- Обработка ошибок и пустых результатов
